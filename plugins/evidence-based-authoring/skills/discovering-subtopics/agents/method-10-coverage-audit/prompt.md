# Method 10 — Coverage Audit

## Role

Independent agent that runs **after agents 01–09 complete**. Reads all nine outputs and writes a structured audit — *not* a flat question list — to a fixed output path with a completion marker.

## Shared scaffold (modified for Agent 10)

```
Topic: {topic}
Depth: {depth_budget}
Run directory: {run_dir}
Output path: {output_path}  (= {run_dir}/outputs/10-coverage-audit.md)
Inputs to read: {run_dir}/outputs/01-*.md through {run_dir}/outputs/09-*.md

Agent 10 uses a STRUCTURED output (not the flat question format of agents 01–09).
Exact section headers below are load-bearing — the Synthesis agent parses them.
The last non-blank line MUST be exactly: <!-- COMPLETE -->
```

## Task

Prevent the illusion of completeness. Audit the aggregate produced by agents 01–09 and produce:
1. **Gaps Found** — dimensions under-covered, with fill-in questions generated by Agent 10
2. **Most Important Questions** — the top 10 across all 9 agents whose answers would most change understanding of the topic
3. **Duplicate Flags** — near-duplicate questions across agents, by agent + question number
4. **Partial Agents** — any of 01–09 whose output lacks the completion marker

## Inputs — what this agent sees

Agent 10 sees **only** the nine outputs from 01–09. It does **NOT** receive:
- The topic description or anchor sources
- Any `reference_sources`
- Any prior aggregation or synthesis

This keeps the audit free of priming.

## Tools

Requires **Read, Write, Glob, Grep**. Does NOT need WebSearch.

## Method

### Step 10.1 — Inventory each of the nine outputs
For each, count total questions, which tags are present, and whether the file ends with `<!-- COMPLETE -->`.

### Step 10.2 — Identify gaps
Compare coverage against these dimensions:
- 5W1H coverage (all six vectors populated across methods 01–02)
- Temporal coverage (past / present / future populated, primarily from 05)
- Scale ladder populated (individual → civilisation, from 05)
- Stakeholder breadth (from 02 and 09)
- Pre-mortem lens completeness (all three lenses in 03)
- Cross-domain breadth (distinct fields in 02 + 08)
- Assumption bedrock depth (06)
- Source slot completeness (07: experts / books / conferences / papers / communities)
- Meta coverage (09: framing / taboo / dissolution / power / epistemology / method-reflection)

For each gap, generate 3–5 questions filling it.

### Step 10.3 — Select the Most Important Questions
Across all questions in agents 01–09 outputs, pick the 10 whose answers would most change understanding. Cite source agent + question number: `(Agent 3, Q7)`.

Selection heuristics (not a single rule):
- The question's answer would force a structural change in how someone thinks about the topic
- The question names a dependency or assumption that, if wrong, invalidates many other questions
- The question surfaces a dimension that the current framing suppresses
- The question appeared from multiple independent methods (cross-validated — see 10.4)

### Step 10.4 — Flag near-duplicates across agents
Walk every question in every agent's output. Flag pairs (or clusters) where the same question is asked in different words. Report as: `Agent {n} Q{x} ≈ Agent {m} Q{y}: {shared phrasing or theme}`.

### Step 10.5 — Partial agents
If any of outputs 01–09 lacks the `<!-- COMPLETE -->` marker, list it. This is a separate section — partial runs are a dispatcher concern, not a coverage gap.

## Verification

1. All nine inputs were actually read — evidence: each appears somewhere in the audit (Gaps / Most Important / Duplicate Flags / Partial Agents).
2. **Most Important Questions** has exactly 10 entries, each tagged with source agent + Q number.
3. **Gaps Found** is populated or explicitly says "None found — all dimensions covered".
4. **Duplicate Flags** is populated or explicitly says "None found".
5. **Partial Agents** is honest — do not hide missing completion markers.
6. Last non-blank line is exactly `<!-- COMPLETE -->`.

## Gotchas

G-01. This agent must NOT read `run.md` topic description, `reference_sources`, or external material beyond the nine outputs. Priming the auditor contaminates the gap analysis.

G-02. The gap-fill questions generated in §Gaps Found are the ONE place Agent 10 generates. For everything else (Most Important, Duplicate Flags) the agent only selects / pairs existing questions.

G-03. **Partial Agents** are not coverage gaps — they are dispatcher problems. Keep the sections separate.

G-04. When Duplicate Flags are dense (> 10% of total questions), that is signal: the agents' sub-techniques overlap heavily on this topic. Note it — Synthesis will merge aggressively.

G-05. Completion marker rules — see [../../references/checkpointing-protocol.md](../../references/checkpointing-protocol.md).
